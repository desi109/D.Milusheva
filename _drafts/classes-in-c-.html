What Is a Class?

A class is a building block of a software application. Applications consist of multiple classes, each responsible for a particular behavior. All these classes together will provide the behavior expected from the application.

Let's take a look at a real-world example. We all are familiar with the concept of blogs and posts. Let's say we would like to implement a web application as a blog engine with typical layered architecture. Our application will consist of at least three layers. Presentation, business logic which also referred to as domain layer, and data access which is also referred to as the persistence layer.

In each of these layers, we can have classes each responsible for something particular. For example, in the presentation layer, we can have a class like a PostView, which is purely responsible for displaying imposed to the user. In the business logic or domain layer, we can have the concept of the Post itself. And finally, in the data access or persistence layer, we can have a class like a PostRepository, which knows how to save as opposed to the database or loading from a database.

A real-world application often consists of hundreds or even thousands of classes collaborating with a shudder. So for you, as a C# programmer, it is absolutely crucial to understand classes and how to use them effectively, in order to build a maintainable, clean, robust and reliable application.

The class essentially has two parts - data which is represented by fields and behavior which is represented by methods or functions. Here is how we illustrate a class in UML or unified modeling language.

So in UML, a class has three parts. The first part is the name of the class, e.g. Person. Then there are fields or attributes of that class, which represent data about that class, e.g. Name, Age, Height, Weight. And finally, we have the methods or functions which provide some behavior, e.g. Walk(), Talk(), Eat(), Sleep().
